Hey, I have gone through your changes, good start but a few things need fixing they are The history endpoint needs guardrails. Right now if someone queries with a huge date range they could get back thousands of snapshots in one response. Add a max range of 365 days (return 400 if exceeded) and cap results to the most recent 100 snapshots within the range. Also make from/to optional default to to today and from to 30 days before to. Rates are fetched on every single recalculation. If a user calls POST /current/wallet/refresh three times in a row, that's three Feign calls to statistics-service. Cache the rates with a short TTL (say 5 minutes) so repeated recalculations reuse the same rates. The changeBaseCurrency endpoint path should be /current/wallet/base-currency, not /current/wallet/currency. "currency" is ambiguous "base-currency" makes it clear this changes the display currency, not adding a new one to the wallet. Clean up the plan file (.claude/plans/concurrent-wandering-pie.md).
