Hey there, currently, the app do not have any built in way to validate form inputs. so, for example if a user types a bad email in a TextInput or maybe pick a invalid option combo or leaves a required field blank, there's nothing in currently to catch it, report it or show the user what went wrong. So, I want you to add widget validation to this app that covers things like, first there should be composable validator primitives, so things like Required, MinLength, MaxLength, Pattern (regex-based) and so on. A widget should be able to have multiple validators that run in order, should return at the first failure, but there should also be a mode to collect all errors at once. next, there should be widget level validation api, the base Widget class needs methods for adding/removing validators, triggering validation (which returns the results) and a quick is_valid() check. next, there should be visual error feedback. also, there should be cross widget validation, support for validators that check relationships between multiple widgets (for example "confirm password matches password"), and next there should be menu level form validation and theme integration. and finally, please extend the widget addition api so that I can pass validators=[...] and required=True directly when adding a widget (example menu.add.text_input("Email:", validators=[Email()], required=True))
